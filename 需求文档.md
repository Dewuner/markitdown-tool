产品需求文档 (PRD)：MarkItDown Desktop v1.1
1. 产品概述
产品定位： 一款主打隐私安全、纯本地运行的跨平台文档转 Markdown 桌面生产力工具。

设计语言： 极致的黑白极简主义（Monochrome），强调内容优先，提供沉浸式的无干扰工作流。

目标用户： 程序员、科研人员、自媒体创作者、学生。

2. 技术架构选型
底层基座： Tauri (Rust) —— 负责原生窗口、文件系统交互、跨进程通信。

前端框架： Vue 3 (Composition API) + Vite。

UI 组件库： shadcn-vue 配合 Tailwind CSS（或选 Naive UI）。选用其默认的 Zinc/纯粹黑白配色方案。

核心引擎： Python (打包为 Sidecar) —— 封装 MarkItDown 库。

本地存储： SQLite —— 存储历史记录与用户配置。

3. 核心功能需求 (Functional Requirements)
3.1 核心转换模块

支持拖拽或选择 .pdf, .docx, .xlsx, .pptx, .html 格式文件。

支持多文件/文件夹批量转换，前端需提供清晰的黑白进度条（Progress Bar）或状态指示器。

具备底层容错机制，跳过损坏文件并在界面反馈。

3.2 界面与交互体验 (黑白 UI 规范)

全局设计： 摒弃多余色彩，以黑、白、灰（不同层级的深浅灰色表示层级关系）为核心。利用字重（Font Weight）和留白（Margin/Padding）来区分信息结构。

AI 风格侧边栏 (历史记录)：

左侧提供常驻侧边栏（可折叠），深色模式下为纯黑，浅色模式下为浅灰。

记录列表采用简约的 Card 设计，显示原文件名与转换状态。

实时对比工作区：

左右分栏设计。中间的分隔线（Splitter）极其细微，鼠标悬浮时加深颜色。

左侧显示源文件摘要，右侧集成只读的 Markdown 渲染器（支持代码高亮，同样采用黑白/灰阶代码主题，如 GitHub Light/Dark）。

3.3 进阶增强功能

图片拓展： 默认提取图片至本地同级 assets 文件夹；预留图床配置项（未来版本）。

AI 智能排版： 界面右上角提供一个低调的“AI 优化”按钮（例如一颗星标 icon），点击后调用本地或云端大模型 API 修复 Markdown 格式。
