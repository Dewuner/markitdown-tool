# MarkItDown Desktop - 开发架构与编码规约

## 1. 核心技术栈与语法限定
- **前端框架**：Vue 3 + Vite。必须使用 Composition API 和 `<script setup>` 语法。
- **UI 框架**：shadcn-vue + Tailwind CSS。样式一律使用 Tailwind 工具类，保持黑白极简风格，除非必要不写自定义 `<style>`。
- **桌面端基座**：Tauri v2。使用 Rust 作为系统交互层。
- **核心逻辑侧**：Python 3.10+ (MarkItDown 库)。将通过 Tauri 的 Sidecar 机制打包调用。

## 2. 目录结构规范
项目遵循以下严格的目录结构：
├── /src # Vue 3 前端代码
│ ├── /components # 纯展示类 UI 组件 (shadcn等)
│ ├── /views # 页面级视图组件
│ ├── /lib # 工具函数 (如 tauri API 封装)
│ └── /assets # 静态资源
├── /src-tauri # Tauri (Rust) 后端代码
└── /python-core # Python 脚本与核心逻辑

## 3. 跨进程通信 (IPC) 规约
- **前端 -> Tauri**：使用 Tauri 官方 API `invoke('command_name', { payload })` 进行调用。
- **Tauri -> Python**：使用 `tauri-plugin-shell` 唤起 Python 子进程。Python 脚本的输出必须严格为 JSON 格式，Tauri 解析 JSON 后返回给前端。
- 错误处理：所有跨界调用必须使用 try-catch 包裹，并向 UI 层返回标准化的错误信息结构：`{ success: boolean, data?: any, error?: string }`。
